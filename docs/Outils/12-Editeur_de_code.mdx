### Vanilla JS
:::info
<img src="https://monwikidevweb.la-loutre.fr/img/outils/vanilla.jpg" style={{ width: '80px', float: 'left', margin:'0 40px 20px 0'}} />
Le terme « Vanilla JS » a été inventé pour des sites Web n'utilisant aucune bibliothèque ou structure, reposant
entièrement sur la fonctionnalité JavaScript standard.

"VanillaJS est un nom auquel se référer en utilisant du JavaScript brut sans aucune bibliothèque supplémentaire comme
jQuery. Les gens l'utilisent comme une blague pour rappeler aux autres développeurs que de nombreuses choses peuvent
être faites de nos jours sans avoir besoin de bibliothèques JavaScript supplémentaires.

- [Vanilla JS](http://vanilla-js.com/)
- [You SHOULD Learn Vanilla JavaScript Before JS Frameworks - Snipcart](https://snipcart.com/blog/learn-vanilla-javascript-before-using-js-frameworks)
- [JavaScript - Wikipedia](https://en.wikipedia.org/wiki/JavaScript#Libraries_and_frameworks)

:::

### Babel
:::info
<img src="https://monwikidevweb.la-loutre.fr/img/outils/babel.svg" style={{ width: '80px', float: 'left', margin:'0 40px 20px 0'}} />
Babel est un compilateur JavaScript.<br />
Babel est une chaîne d'outils qui est principalement utilisée pour convertir le code ECMAScript 2015+ en une
version rétrocompatible de JavaScript dans les navigateurs ou environnements actuels et anciens. Voici les
principales choses que Babel peut faire pour vous :
- Transformer la syntaxe
- Fonctionnalités Polyfill manquantes dans votre environnement cible (via un polyfill tiers tel que core-js)
- Transformations de code source (codemods)

- [Babel · The compiler for next generation JavaScript](https://babeljs.io/)
:::

### ESLint

:::info

<img src="https://monwikidevweb.la-loutre.fr/img/outils/ESLint.png" style={{ width: '80px', float: 'left', margin:'0 40px 20px 0'}} />

ESLint est un outil d'analyse de code statique pour identifier les modèles problématiques trouvés dans le code JavaScript.
Il a été créé par Nicholas C. Zakas en 2013. Les règles d'ESLint sont configurables et des règles personnalisées peuvent
être définies et chargées. ESLint couvre à la fois les problèmes de qualité du code et de style de codage. ESLint prend
en charge les normes actuelles d'ECMAScript et la syntaxe expérimentale des projets de normes futures. Le code utilisant
JSX ou TypeScript peut également être traité lorsqu'un plugin ou un transpilateur est utilisé.

- [ESLint - Pluggable JavaScript linter](https://eslint.org/)
- [ESLint - Wikipedia](https://en.wikipedia.org/wiki/ESLint)

:::
