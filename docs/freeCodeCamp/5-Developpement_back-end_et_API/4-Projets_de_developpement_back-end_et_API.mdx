# Projets API et microservices

## Introduction aux Projets API et microservices

[APIs and Microservices Projects | freeCodeCamp.org](https://www.freecodecamp.org/learn/apis-and-microservices/#apis-and-microservices-projects)

Vous avez déjà travaillé avec des API, mais maintenant que vous connaissez npm, Node, Express, MongoDB
et Mongoose, il est temps de créer les vôtres. Tirez parti de tout ce que vous avez appris jusqu'à
présent pour créer 5 microservices différents, qui sont des applications plus petites dont la portée
est limitée.

Après les avoir créés, vous disposerez de 5 API de microservices intéressantes que vous pourrez montrer
à vos amis, à votre famille et à des employeurs potentiels. Oh, et vous aurez également une nouvelle
certification des API et des microservices.

## 1. Microservice d'horodatage

[APIs and Microservices Projects - Timestamp Microservice | Learn | freeCodeCamp.org](https://www.freecodecamp.org/learn/apis-and-microservices/apis-and-microservices-projects/timestamp-microservice)

Créez une application JavaScript full stack qui est fonctionnellement similaire à celle-ci :
https://timestamp-microservice.freecodecamp.rocks/. Travailler sur ce projet vous impliquera
d'écrire votre code en utilisant l'une des méthodes suivantes :
- Clonez [ce dépôt GitHub](https://github.com/freeCodeCamp/boilerplate-project-timestamp/) et
terminez votre projet localement.
- Utilisez [notre projet de démarrage repl.it](https://repl.it/github/freeCodeCamp/boilerplate-project-timestamp)
pour terminer votre projet.
- Utilisez un constructeur de site de votre choix pour terminer le projet. Assurez-vous d'incorporer
tous les fichiers de notre dépôt GitHub.

Lorsque vous avez terminé, assurez-vous qu'une démo fonctionnelle de votre projet est hébergée dans
un endroit public. Envoyez-lui ensuite l'URL dans le champ Lien `Solution Link`. Vous pouvez
également soumettre un lien vers le code source de vos projets dans le champ `GitHub Link`.

**#1 :** Vous devez fournir votre propre projet, pas l'exemple d'URL.

**#2 :** Une demande à `/api/timestamp/:date?` avec une date valide doit retourner un objet JSON avec une
clé `unix` qui est un horodatage Unix de la date d'entrée en millisecondes.

**#3 :** Une demande à `/api/timestamp/:date?` avec une date valide doit renvoyer un objet JSON avec une
clé `utc` qui est une chaîne de la date d'entrée au format : `Thu, 01 Jan 1970 00:00:00 GMT`.

**#4 :** Une requête à `/api/timestamp/1451001600000` doit renvoyer `{ unix: 1451001600000, utc: 'Fri,
25 Dec 2015 00:00:00 GMT' }`.

**#5 :** Votre projet peut gérer des dates qui peuvent être analysées avec succès par `new Date(date_string)`.

**#6 :** Si la chaîne de date d'entrée n'est pas valide, l'API renvoie un objet ayant la structure
`{ error: 'Invalid Date' }`.

**#7 :** Un paramètre de date vide doit renvoyer l'heure actuelle dans un objet JSON avec une clé `unix`.

**#8 :** Un paramètre de date vide doit renvoyer l'heure actuelle dans un objet JSON avec une clé `utc`.

## 3. Microservice de réducteur d'URL

[APIs and Microservices Projects - URL Shortener Microservice | Learn | freeCodeCamp.org](https://www.freecodecamp.org/learn/apis-and-microservices/apis-and-microservices-projects/url-shortener-microservice)

Créez une application JavaScript full stack qui est fonctionnellement similaire à celle-ci :
https://url-shortener-microservice.freecodecamp.rocks/. Travailler sur ce projet vous impliquera
d'écrire votre code en utilisant l'une des méthodes suivantes :
- Clonez [ce dépôt GitHub](https://github.com/freeCodeCamp/boilerplate-project-urlshortener/)
et terminez votre projet localement.
- Utilisez [notre projet de démarrage repl.it](https://repl.it/github/freeCodeCamp/boilerplate-project-urlshortener)
pour terminer votre projet.
- Utilisez un constructeur de site de votre choix pour terminer le projet. Assurez-vous d'incorporer
tous les fichiers de notre dépôt GitHub.

Lorsque vous avez terminé, assurez-vous qu'une démo fonctionnelle de votre projet est hébergée dans un
endroit public. Envoyez-lui ensuite l'URL dans le champ `Solution Link`. Vous pouvez également soumettre
un lien vers le code source de vos projets dans le champ `GitHub Link`.

**CONSEIL :** n'oubliez pas d'utiliser un middleware d'analyse du corps pour gérer les requêtes POST.
En outre, vous pouvez utiliser la fonction `dns.lookup(host, cb)` du module principal `dns` pour vérifier
une URL soumise.

**#1 :** Vous devez fournir votre propre projet, pas l'exemple d'URL.

**#2 :** Vous pouvez POST une URL vers `/api/shorturl/new` et obtenir une réponse JSON avec les propriétés
`original_url` et `short_url`. Voici un exemple : `{ original_url: 'https://freeCodeCamp.org', short_url: 1 }`.

**#3 :** Lorsque vous visitez `/api/shorturl/<short_url>`, vous serez redirigé vers l'URL d'origine.

**#4 :** Si vous transmettez une URL non valide qui ne respecte pas le format `http://www.example.com` valide,
la réponse JSON contiendra `{ error: 'invalid url' }`.

:::note guide : Microservice de réducteur d'URL
### Explication du problème

[freeCodeCamp Challenge Guide: URL Shortener Microservice - Guide - The freeCodeCamp Forum](https://forum.freecodecamp.org/t/freecodecamp-challenge-guide-url-shortener-microservice/301509)

Les principales fonctionnalités pour réaliser cet exercice sont la création et la récupération d'URL à partir
de la base de données.

### Conseils
#### Créer une URL courte

- Connectez-vous à votre instance de base de données.<br />
**Remarque :** Il est important de vérifier l'état de votre connexion Mongoose avant de vous plonger dans le
problème, juste pour vérifier si tout va bien avec la configuration de votre base de données. Cela devrait aider :
`mongoose.connection.readyState`.
- Recevez une requête POST contenant une URL à enregistrer sur la base de données.
- Vérifiez s'il s'agit d'une URL valide à l'aide de `dns.lookup(url, callback)`.
    - N'oubliez pas que vous devez importer le module DNS avec les fichiers requis.
- Générez une sorte d'identifiant pour enregistrer votre URL d'origine dans la base de données.
    - Un hachage SHA-1 peut être utilisé, ou même l'ID d'objet lors de l'enregistrement de l'élément.
    - Il existe de nombreux exemples sur Internet sur la façon de générer une sorte d'identifiant, d'essayer de
    l'explorer ou de créer le vôtre.
    - Un exemple de ce à quoi cela devrait ressembler : `{'url': www.freecodecamp.org, 'hash': 'ef49fa8b4'}`.

#### Récupération d'une URL courte

- Recevez une requête GET contenant un identifiant utilisé pour trouver une URL stockée.
- Essayez de trouver une URL enregistrée pour cet identifiant.
- Redirigez l'utilisateur vers l'URL.<br />
**Remarque :** La fonction `res.redirect(url)` a besoin que l'url donnée ait un protocole défini (http://, https://),
ou elle la concaténera simplement en tant qu'extension de votre domaine actuel. Par exemple :<br />
   - bonne URL : https://www.freecodecamp.org,
   - mauvaise URL : www.freecodecamp.org.
- N'oubliez pas de gérer les situations d'erreur avec une réponse appropriée comme, `res.json({'error':'invalid URL'});`
:::

## 4. Suivi d'exercices

[APIs and Microservices Projects - Exercise Tracker | Learn | freeCodeCamp.org](https://www.freecodecamp.org/learn/apis-and-microservices/apis-and-microservices-projects/exercise-tracker)

Créez une application JavaScript full stack qui est fonctionnellement similaire à celle-ci :
https://exercise-tracker.freecodecamp.rocks/. Travailler sur ce projet vous impliquera d'écrire votre code
en utilisant l'une des méthodes suivantes :
- Clonez [ce dépôt GitHub](https://github.com/freeCodeCamp/boilerplate-project-exercisetracker/) et terminez votre
projet localement.
- Utilisez [notre projet de démarrage repl.it](https://repl.it/github/freeCodeCamp/boilerplate-project-exercisetracker)
pour terminer votre projet.
- Utilisez un constructeur de site de votre choix pour terminer le projet. Assurez-vous d'incorporer tous les
fichiers de notre dépôt GitHub.

Lorsque vous avez terminé, assurez-vous qu'une démo fonctionnelle de votre projet est hébergée dans un endroit public.
Envoyez-lui ensuite l'URL dans le champ `Solution Link`. Vous pouvez également soumettre un lien vers le code source
de votre projet dans le champ `GitHub Link`.

**#1 :** Vous devez fournir votre propre projet, pas l'exemple d'URL.

**#2 :** Vous pouvez `POST` sur `/api/exercice/new-user` avec les données de formulaire `username` pour créer un nouvel
utilisateur. La réponse retournée sera un objet avec les propriétés `username` et `_id`.

**#3 :** Vous pouvez faire une requête `GET` à `api/exercice/users` pour obtenir un tableau de tous les utilisateurs.
Chaque élément du tableau est un objet contenant `username et `_id` d'un utilisateur.

**#4 :** Vous pouvez `POST` dans `/api/exercice/add` avec les données de formulaire `userId=_id`, `description`,
`duration` et éventuellement `date`. Si aucune date n'est fournie, la date actuelle sera utilisée. La réponse renvoyée
sera l'objet utilisateur avec les champs d'exercice ajoutés.

**#5 :** Vous pouvez faire une requête `GET` à `/api/exercice/log` avec un paramètre de `userId=_id` pour récupérer un
journal d'exercice complet de n'importe quel utilisateur. La réponse renvoyée sera l'objet utilisateur avec un tableau
`log` de tous les exercices ajoutés. Chaque élément de journal a les propriétés `description`, `duration` et `date`.

**#6 :** Une demande au journal d'un utilisateur (`/api/exercice/log`) renvoie un objet avec une propriété `count`
représentant le nombre d'exercices renvoyés.

**#7 :** Vous pouvez ajouter des paramètres `from`, `to` et `limit` à une requête `/api/exercice/log` pour récupérer
une partie du journal de n'importe quel utilisateur. `from` et `to` sont des dates au format `yyyy-mm-dd`. `limit`
est un entier du nombre de journaux à renvoyer.
