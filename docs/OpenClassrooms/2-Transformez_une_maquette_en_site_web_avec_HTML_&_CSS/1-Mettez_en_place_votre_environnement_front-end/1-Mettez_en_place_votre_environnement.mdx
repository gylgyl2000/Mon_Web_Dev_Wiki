## Installez vos outils de développement front-end

[Installez vos outils de développement front-end - Mettez en place votre environnement front-end - OpenClassrooms](https://openclassrooms.com/fr/courses/6943241-mettez-en-place-votre-environnement-front-end/6943248-installez-vos-outils-de-developpement-front-end)

<br />

Dans ce chapitre, on va installer :
- l'un des environnements de développement gratuits les plus utilisés : Visual Studio Code ;
- les deux navigateurs les plus répandus : Chrome et Firefox.

:::info
Si vous ne savez pas encore ce qu'est un environnement de développement, ou pourquoi on a besoin de plusieurs navigateurs, ne
vous inquiétez pas ! On en parlera en détail dans les prochains chapitres. Ici, on va simplement s'assurer de les télécharger
et de les installer correctement, et on les découvrira ensuite.
:::

### Installez Visual Studio Code

![](/img/OpenClassrooms/16073346800551_P1C1-1_Intallez_VSCode.png)

Le premier outil qu'on va installer est Visual Studio Code, ou _VS Code_, qui sera votre outil de travail principal.

Téléchargez d'abord l'installateur depuis le site officiel : [https://code.visualstudio.com](https://code.visualstudio.com).

:::note !
Normalement, votre système d'exploitation (Windows ou macOS) sera sélectionné par défaut. Si ce n'est pas le cas, sélectionnez-le
dans le menu déroulant, et sélectionnez l'installation **Stable**.
:::

#### Sous **Windows**

Une fois le téléchargement terminé, lancez l'installateur depuis son dossier de téléchargement, et acceptez le _User Agreement_.
Vous pouvez avancer dans les différents écrans de l'installation sans modifier la configuration par défaut.

![Accept the license agreement](/img/OpenClassrooms/16085651860482_image29.png)

Acceptez le contrat de licence

![Les options par défaut](/img/OpenClassrooms/1608565203072_image64.png)

Les options par défaut

![Prêt à installer](/img/OpenClassrooms/16085652164701_image50.png)

Prêt à installer

:::note
Après l'installation, vous avez le choix ou non de lancer VS Code, mais faites-le uniquement si vous avez terminé les installations
et que vous êtes prêt à avancer au prochain chapitre !
:::

#### Sous **macOS**

Le fichier que vous téléchargez est l'application déjà prête à l'emploi. Glissez-la dans votre dossier _Applications_ pour la
rendre accessible depuis le _launchpad_.

![Glissez VS Code dans Applications](/img/OpenClassrooms/16085653980659_image15.png)

Glissez VS Code dans Applications

Et voilà ! VS Code est installé correctement : il est temps d'installer les navigateurs.

:::note
Si vous avez déjà installé Chrome et/ou Firefox, n'hésitez pas à passer les instructions correspondantes.
:::

### Installez vos navigateurs

Les différents navigateurs n'affichent pas forcément les sites et applications exactement de la même manière, et il faut souvent
modifier son code pour qu'il soit compatible. Il vous sera donc indispensable d'avoir accès à plusieurs navigateurs, en tant que
développeur front-end, pour tester vos projets. Il faudra au moins les deux navigateurs les plus répandus : Chrome et Firefox.

#### Installez Chrome

![](/img/OpenClassrooms/16073347151941_P1C1-2_Installez_Chrome.png)

Allez sur [https://www.google.com/chrome/](https://www.google.com/chrome/) et téléchargez Google Chrome pour votre système d'exploitation.

##### Sous **Windows**

Lancez l'installateur téléchargé, l'installation se fait automatiquement.

##### Sous **macOS**

Ouvrez le fichier que vous avez téléchargé. Glissez l'icône Chrome dans le dossier _Applications_ dans la fenêtre qui s'ouvre.

![Glissez Chrome dans Applications](/img/OpenClassrooms/16085652323839_image3.png)

Glissez Chrome dans Applications

#### Installez Firefox

![](/img/OpenClassrooms/16073347303794_P1C1-3_Installez_Firefox.png)

Naviguez vers [https://www.mozilla.org/fr/firefox/new/](https://www.mozilla.org/fr/firefox/new/) et téléchargez la dernière version de Firefox pour votre système d'exploitation.

##### Sous **Windows**

Lancez l'installateur téléchargé, l'installation se fait automatiquement.

##### Sous **macOS**

Ouvrez le fichier téléchargé. Glissez l'icône Firefox dans le dossier Applications dans la fenêtre qui s'ouvre.

![Glissez Firefox dans Applications](/img/OpenClassrooms/16085652466189_image55.png)

Glissez Firefox dans Applications

L'autre différence se situe au niveau des **raccourcis clavier**, et je donnerai toujours les raccourcis des deux plateformes.

À gauche de la fenêtre VS Code, le contenu du dossier de code apparaît. On peut maintenant fermer l'onglet d'accueil, et ouvrir

![A JavaScript file](/img/OpenClassrooms/16042519487563_Screenshot_2020-11-01_at_18.31.19.png)

Un fichier JavaScript

Là vous voyez un fichier écrit en **JavaScript**, l'un des nombreux langages gérés par défaut par VS Code. Ne vous inquiétez pas
si vous ne comprenez pas le code, ceci n'est pas un cours de JavaScript ! Je voudrais simplement attirer votre attention sur
l'onglet **Editor** qui s'est ouvert.

La première chose que l'on peut constater, c'est que le texte est **coloré**. Les différentes couleurs correspondent à différents
éléments du langage utilisé, rendant votre code plus facile à lire et à comprendre.

Vous constaterez ensuite que les lignes du fichier sont **numérotées**. C'est une aide qui, vous le verrez, est indispensable,
car elle nous permet de retrouver rapidement les erreurs, ainsi que de collaborer bien plus facilement avec d'autres développeurs.

:::info
Souvent, quand un programme que nous créons rencontre une erreur, il nous fournit le nom du fichier et le numéro de la ligne où
l'erreur se trouve !
:::

Passez maintenant votre souris entre les numéros de ligne et le code : vous y verrez apparaître des flèches qui permettent de
**réduire** le code. Cliquez sur les flèches des lignes **4**, **20**, **31,** et **35** :

![Réduire du code](/img/OpenClassrooms/1604252580551_Screenshot_2020-11-01_at_18.42.13.png)

Réduire du code

Le code est toujours là (les numéros de ligne reflètent la vérité cachée !), mais le fait de cacher certaines sections du code
rend l'écran **moins encombré** et peut nous aider à nous concentrer sur l'essentiel.

### Gérez vos dossiers de travail

![](/img/OpenClassrooms/16073347873366_P1C2-2_Gerez_vos_dossiers_de_travail.png)

Gérez ici les extensions de fichiers et organisez vos dossiers. Cela vous sera bien utile pour gérer vos projets !

<Vimeo vimeoId="491648267" />

<br />

En travaillant avec l'éditeur de code, on a vu qu'on pouvait ouvrir un fichier en cliquant dessus dans l'explorateur de fichiers
à gauche de l'écran. Cet explorateur vous permet également de **gérer vos fichiers et dossiers de travail**. Si vous y passez
votre souris, vous verrez apparaître quelques icônes dans l'en-tête :

![Opérations de dossier](/img/OpenClassrooms/16085656856691_image17.png)

Opérations de dossier

Le premier vous permet de **créer un nouveau fichier**. Quand on crée un fichier, on doit spécifier son **extension**.

:::info
L'extension d'un fichier nous spécifie son **type**, et ce qu'on pourra y trouver. C'est ce qui se trouve après le dernier point
dans un nom de fichier ; par exemple, index._html_, blog._js_, styles._css_…
:::

VS Code interprète l'extension du fichier pour colorer correctement le code et pour mettre en place la bonne autocorrection, par
exemple. Cliquez sur l'icône `New File`, et essayez ces trois exemples : `new.html`, `new.js`, `new.css`. Vous voyez que VS Code
reconnaît l'extension, grâce à l'icône qu'il y attribue.

![Nouveaux fichiers](/img/OpenClassrooms/16085656968735_image16.png)

Reconnaissance des extensions des nouveaux fichiers

:::info
Cette différenciation par icône va faciliter la navigation !
:::

La deuxième icône de l'en-tête vous permet de **créer des dossiers** et sous-dossiers. On différencie les dossiers dans
l'explorateur par l'icône _chevron_. Quand vous cliquez sur un dossier, il s'ouvre pour montrer son contenu. Créez un
sous-dossier `icons` au dossier `images` en le sélectionnant, puis en cliquant sur `New Folder` :

![Nouveau dossier](/img/OpenClassrooms/16085657060202_image20.png)

Nouveau dossier

La prochaine icône, la flèche en cercle, permet de **rafraîchir** l'explorateur. Cette fonctionnalité vous sera utile notamment
quand des fichiers auront été créés par un autre outil, et que VS Code devra en prendre compte.

La dernière icône est un raccourci pour **replier** tous les dossiers qui sont ouverts dans l'explorateur. Ça vous sera utile
quand vous aurez des projets plus complexes avec beaucoup de dossiers, comme ce projet front-end avancé :

![Une multitude de dossiers...](/img/OpenClassrooms/16085657188725_image7.png)

Une multitude de dossiers...

:::info
En tout cas, l'explorateur de fichiers intégré à VS Code vous permettra de vous organiser et de naviguer facilement à l'intérieur
de votre projet.
:::

### Prenez en main la palette de commandes

![](/img/OpenClassrooms/16073348090971_P1C2-3_Prenez_en_main_la_palette_de_commandes.png)

La palette de commande est une fonctionnalité dont vous ne pourrez plus vous passer ! Elle vous sera d'une grande aide pour
travailler plus efficacement sur VS Code. Découvrez là dans ce sceencast :

<Vimeo vimeoId="491648321" />

<br />

Dans VS Code, il existe un raccourci qui vous permet de trouver toutes les commandes possibles et imaginables en tapant leur nom.
Ça veut dire que vous n'avez pas besoin d'apprendre par cœur dans quel menu se trouve une commande, ni son raccourci clavier
(même si vous finirez par en apprendre certains !). Tout ça grâce à la **palette de commandes**, que vous ouvrez avec `Ctrl+Maj+P`
sur PC, ou `Cmd+Maj+P` sur Mac :

![La palette de commandes](/img/OpenClassrooms/16047517833775_Screenshot_2020-11-07_at_13.22.10.png)

La palette de commandes

Par exemple, imaginons que vous ayez plusieurs éditeurs ouverts, et que vous vouliez tous les fermer pour travailler sur une autre
fonctionnalité de votre projet. Ouvrez la palette de commandes et tapez simplement le mot "Close" :

![Fermer tous les éditeurs](/img/OpenClassrooms/16047519832114_Screenshot_2020-11-07_at_13.25.27.png)

Fermer tous les éditeurs

Vous y voyez la commande "Close All Editors", ou "Fermer tous les éditeurs". Vous pouvez y accéder avec les flèches de votre
clavier, et appuyer sur "Entrée" pour exécuter la commande.

:::info
Notez aussi que la palette de commandes vous montre également le raccourci clavier pour la commande que vous souhaitez exécuter !
:::

Un autre exemple de commande rapide : la création d'un nouveau fichier. Ouvrez la palette de commandes et tapez "New File" :

![New File](/img/OpenClassrooms/16047522778353_Screenshot_2020-11-07_at_13.30.38.png)

Nouveau fichier

Cela accomplit la même chose que lorsque vous cliquez sur le bouton dans l'explorateur de fichiers, mais **sans utiliser votre
souris**. Tout geste qui requiert la souris nous ralentit : essayez de garder le plus possible les mains sur le clavier !

:::info
La palette de commandes est un peu l'arme secrète de VS Code : elle vous facilitera l'apprentissage de l'outil, et vous permettra
d'être plus efficace !
:::

(aussi connu sous les noms de _shell_, _PowerShell_, _invite de commandes_…). C'est pour cela que VS Code permet d'utiliser

### Utilisez Git

![](/img/OpenClassrooms/16073349683126_P1C2-5_Utiliser_Git.png)

:::info
Si vous ne connaissez pas comment utilisez Git, je vous invite à suivre le cours "[utilisez Git et GitHub pour vos projets de développement](https://openclassrooms.com/fr/courses/5641721-utilisez-git-et-github-pour-vos-projets-de-developpement)".
:::

Dans ce screencast, vous allez découvrir comment VS Code intègre les fonctionnalités de Git.

<Vimeo vimeoId="491648453" />

<br />

Comme vous le savez, le système de contrôle de version Git est extrêmement important dans le monde du développement, que ce soit
au niveau individuel, au niveau d'une équipe, ou même au niveau d'une entreprise. VS Code intègre donc tous les outils nécessaires
à son utilisation.

#### Clonez un dépôt

Tout d'abord, vous pouvez cloner un dépôt Git directement depuis VS Code. Pour faire cela, fermez tout projet actuellement ouvert,
et ouvrez l'explorateur de fichiers (en haut à gauche) :

![Clone a repo](/img/OpenClassrooms/16047669171213_Screenshot_2020-11-07_at_17.33.51.png)

Cloner un dépôt

Vous pouvez ensuite cliquer sur "Clone Repository", puis coller l'URL du dépôt à cloner ; par exemple celui que nous avons
utilisé jusqu'ici :

[https://github.com/OpenClassrooms-Student-Center/5543061-ecrivez-du-javascript-pour-le-web-activity.git](https://github.com/OpenClassrooms-Student-Center/5543061-ecrivez-du-javascript-pour-le-web-activity.git)

Quand vous validez l'URL, VS Code vous propose de choisir l'emplacement où vous souhaitez cloner le code, et vous propose d'ouvrir
le dépôt cloné. En cliquant sur "Open", vous vous retrouvez dans le bon projet.

#### Créez une branche

Il est temps de travailler sur votre code, et vous souhaitez créer une branche de travail. Les outils Git se trouvent à gauche :

![Les outils Git](/img/OpenClassrooms/16047675764325_Screenshot_2020-11-07_at_17.45.06.png)

Les outils Git

Vous pouvez ouvrir le menu `…` ; allez sur Branch -> Create Branch.

:::info
Bien sûr, vous pouvez aussi chercher "Create Branch" dans la palette de commandes !
:::

Choisissez un nom pour votre branche et appuyez sur Entrée. Vous verrez alors la branche sur laquelle vous vous trouvez, en bas
à gauche de la fenêtre principale :

![La branche active](/img/OpenClassrooms/16085659413319_image18.png)

La branche active

En cliquant sur cette branche, vous pouvez choisir de changer de branche ou d'en créer une nouvelle.

#### Commit

Effectuons quelques modifications. Ouvrez l'explorateur de fichiers, puis ouvrez `blog.html`, et changez la langue `en` en ligne 2
à `fr`. Dans `functions.js`, à la ligne 17, changez "Veuillez saisir quelque chose !" en "Il faut saisir quelque chose !".

:::info
N'oubliez pas d'enregistrer vos modifications à chaque fois avec `Ctrl+S` sur PC ou `Cmd+S` sur Mac !
:::

Rouvrez les outils Git, et vous verrez le résumé de vos modifications. Cliquez sur un fichier pour voir le détail :

![Git Diff](/img/OpenClassrooms/16047681121861_Screenshot_2020-11-07_at_17.54.29.png)

Git Diff

Plusieurs options s'ouvrent donc à nous, mais disons que nous souhaitons _commit_ nos modifications. Il faut d'abord **stage**
nos modifications en cliquant sur les boutons + sur chaque fichier. Puis, entrez simplement votre **message de commit** ("Changed
language", par exemple), et faites Ctrl+Entrée sur PC, ou Cmd+Entrée sur Mac pour effectuer votre commit.

Toutes les commandes liées au contrôle de version se trouvent dans cette partie de VS Code (et sont trouvables par la palette de
commandes), donc n'hésitez pas à les essayer avec vos propres dépôts.

:::info
Vous pouvez même y lier votre compte GitHub !
:::

### Exploitez les extensions

![](/img/OpenClassrooms/16073349891466_P1C2-6_Exploitez_les_plugins.png)

Pour personnaliser encore plus votre configuration de VS Code, vous pouvez installer des **plugins**, ou **_extensions_**. Ces
plugins viennent rajouter des fonctionnalités ou des outils supplémentaires à votre VS Code.

Vous pouvez parcourir toutes les extensions disponibles sur [https://marketplace.visualstudio.com](https://marketplace.visualstudio.com),
mais voici deux extensions que je peux vous conseiller :
- _Color Picker_ – cette extension vous permet de générer des codes couleurs à partir d'une color wheel, facilitant l'utilisation
des couleurs dans vos projets ;
- _open in browser_ – permet d'ouvrir le fichier sélectionné directement dans votre navigateur par défaut, ce qui facilite les

:::info
Le code que vous voyez dans l'inspecteur est du code **HTML** – le langage qui définit la **structure** d'une page web.

Une page web est composée d'**éléments** – du texte, les images, les boutons… ce sont tous des exemples d'éléments. Un élément

enfants : le logo OpenClassrooms, des liens, un bouton Rechercher, et les boutons liés à la connexion.
